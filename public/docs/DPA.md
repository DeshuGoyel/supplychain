# Data Processing Agreement (DPA)

**Effective Date:** [Date]  
**Last Updated:** [Date]

## 1. Introduction

This Data Processing Agreement ("DPA") governs the processing of Personal Data by Supply Chain AI Control Assistant ("Processor") on behalf of its customers ("Controller") in connection with the use of our SaaS platform.

## 2. Definitions

- **"Personal Data"**: Any information relating to an identified or identifiable natural person as defined in applicable data protection laws.
- **"Processing"**: Any operation performed on Personal Data, including collection, recording, organization, storage, alteration, retrieval, use, disclosure, restriction, erasure, or destruction.
- **"Controller"**: The entity that determines the purposes and means of processing Personal Data.
- **"Processor"**: Supply Chain AI Control Assistant, which processes Personal Data on behalf of the Controller.
- **"Sub-processor"**: Any third party engaged by the Processor to process Personal Data.
- **"Services"**: The SaaS platform and related services provided by the Processor.

## 3. Scope and Applicability

### 3.1 Coverage
This DPA applies to all processing of Personal Data by the Processor in connection with:
- Supply chain management features
- User account management
- Billing and subscription processing
- Customer support services
- Analytics and reporting features

### 3.2 Customer Obligations
The Controller represents and warrants that:
- They have the lawful basis to process Personal Data
- The Personal Data is collected and processed in compliance with applicable laws
- They have obtained all necessary consents for processing
- The Personal Data is accurate and up-to-date

## 4. Processing Details

### 4.1 Subject Matter
The subject matter of the data processing is the provision of supply chain management software services.

### 4.2 Duration
Processing will continue for the duration of the service agreement between the parties.

### 4.3 Nature and Purpose
- Account creation and management
- Supply chain data processing and analysis
- User authentication and authorization
- Billing and payment processing
- Customer support and communication
- Service improvement and analytics

### 4.4 Categories of Data Subjects
- Company employees and users
- Supplier contacts
- Customer representatives
- Other individuals involved in supply chain operations

### 4.5 Categories of Personal Data
- **Identity Data**: Name, email, job title, phone number
- **Account Data**: Username, password (hashed), security questions
- **Business Data**: Company information, business address, industry
- **Supply Chain Data**: Supplier contacts, order information, inventory data
- **Technical Data**: IP address, browser type, device information
- **Usage Data**: Feature usage, pages visited, time spent in application

## 5. Processor Obligations

### 5.1 Compliance with Laws
The Processor will:
- Process Personal Data only on documented instructions from the Controller
- Ensure that persons authorized to process Personal Data are subject to confidentiality obligations
- Implement appropriate technical and organizational measures to ensure security
- Respect the conditions for engaging sub-processors
- Assist the Controller with data subject rights requests
- Delete or return Personal Data at the end of the service

### 5.2 Security Measures
The Processor implements appropriate technical and organizational measures including:
- Encryption of Personal Data in transit and at rest
- Regular security assessments and penetration testing
- Access controls and authentication mechanisms
- Network security and intrusion detection
- Employee training and background checks
- Incident response procedures
- Regular backup and recovery processes

### 5.3 Data Breach Notification
The Processor will notify the Controller without undue delay (within 72 hours) of:
- Any Personal Data breach
- Security incidents affecting Personal Data
- Unauthorized access to Personal Data

The notification will include:
- Nature of the breach
- Categories and approximate number of data subjects affected
- Likely consequences of the breach
- Measures taken or proposed to address the breach

- Other countries where sub-processors operate

### 7.2 Adequate Protection
The Processor ensures adequate protection through:

## 8. Data Subject Rights

### 8.1 Processor Assistance
The Processor will assist the Controller in responding to data subject requests including:
- Access requests
- Rectification requests
- Erasure requests
- Data portability requests
- Restriction of processing requests
- Objection to processing

### 8.2 Response Timeline
The Processor will respond to data subject requests within 30 days of receipt.

### 8.3 Technical Assistance
The Processor provides technical means for data subjects to exercise their rights through:
- User account settings

## 9. Audit and Compliance

### 9.1 Audit Rights
The Controller has the right to audit the Processor's data processing activities through:
- Annual compliance reports (SOC 2, ISO 27001)
- Third-party security assessments
- On-site audits (with reasonable notice)
- Remote security assessments

### 9.2 Compliance Certifications
The Processor maintains the following certifications:
- SOC 2 Type II
- ISO 27001
- GDPR compliance certification
- Industry-specific compliance where applicable

### 9.3 Documentation
The Processor maintains documentation of:
- Data processing activities
- Security measures implemented
- Sub-processor relationships
- Data subject rights procedures

## 10. Data Retention and Deletion

### 10.1 Retention Periods
- **Active Accounts**: Personal Data retained while account is active
- **Inactive Accounts**: 90 days after account termination
- **Audit Logs**: 2 years for security and compliance
- **Billing Records**: 7 years for tax and accounting requirements

### 10.2 Deletion Procedures
The Processor will:
- Delete Personal Data within specified timeframes
- Use secure deletion methods to prevent data recovery
- Provide confirmation of deletion upon request
- Maintain deletion logs for audit purposes

### 10.3 Backup Deletion
Personal Data will be deleted from backup systems within 90 days of account termination.

## 11. Return of Data

### 11.1 Data Export
Upon termination of services, the Controller may request:
- Export of Personal Data in machine-readable format
- Secure transfer of data to Controller or designated third party
- Documentation of data deletion procedures

### 11.2 Export Formats
Data can be exported in the following formats:
- JSON
- CSV
- Excel
- XML

### 11.3 Export Timeline
Data exports will be provided within 30 days of request.

## 12. Incident Response

### 12.1 Incident Classification
Security incidents are classified as:
- **Critical**: Data breach or major system compromise
- **High**: Unauthorized access to Personal Data
- **Medium**: Security vulnerabilities
- **Low**: Minor security incidents

### 12.2 Response Procedures
The Processor maintains documented incident response procedures including:
- Incident detection and reporting
- Containment and mitigation measures
- Communication protocols
- Post-incident analysis and improvement

### 12.3 Forensic Investigation
In case of security incidents, the Processor may conduct forensic investigations while respecting the Controller's data protection obligations.

## 13. Liability and Indemnification

### 13.1 Processor Liability
The Processor's liability for damages resulting from data processing is limited to:
- Direct damages up to the amount paid for services in the 12 months preceding the claim
- Excluding liability for indirect, consequential, or punitive damages

### 13.2 Force Majeure
Neither party will be liable for delays or failures in performance due to circumstances beyond their reasonable control.

## 14. Term and Termination

### 14.2 Termination

### 14.3 Survival
